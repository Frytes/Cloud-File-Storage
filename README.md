**Cloud File Storage** — это надежное масштабируемое облачное хранилище файлов с веб-интерфейсом  
(аналог Google Drive / Яндекс.Диск).

Проект разработан с упором на промышленные стандарты:  
**stateless-архитектура**, **асинхронная обработка тяжелых задач**, **CI/CD** и **пирамида тестирования**  
(Unit, Integration, E2E).

---

## 🌟 Ключевые возможности

### 🔐 Аутентификация и Безопасность
- **Локальная регистрация** (с хешированием паролей BCrypt) и авторизация.
- **OAuth 2.0** — вход в один клик через Google.
- **Управление сессиями** — единая сессия через Spring Session + Redis.
- **Изоляция данных** — пользователи имеют доступ только к своему выделенному пространству.

### 📁 Управление файлами (MinIO S3)
- **Stateless подход:** база данных (PostgreSQL) хранит *только* пользователей.  
  Вся структура папок и файлов строится "на лету" из префиксов объектов в S3 хранилище.
- Загрузка файлов и целых папок (Drag & Drop).
- Создание виртуальных директорий, переименование, перемещение (включая вложенные папки) и удаление.
- Мгновенный поиск файлов по названию.

### ⚡ Асинхронная загрузка (RabbitMQ + WebSockets)
- **Гибридное скачивание папок:**
    - Маленькие папки (< 100 МБ) собираются в ZIP-архив синхронно "на лету" (Streaming)  
      без сохранения на диск сервера.
    - Большие папки (≥ 100 МБ) ставятся в очередь **RabbitMQ**.  
      Воркер в фоновом режиме собирает архив и кладет его во временный бакет.
- **Уведомления в реальном времени:** Клиент получает push-уведомление о готовности архива  
  через **WebSockets (STOMP)** и ссылку (Presigned URL) на скачивание.

## 🛠 Технологический стек

### Backend Core
- **Java 21**, **Spring Boot 3** (Web, Security, Data JPA, OAuth2 Client, WebSockets, AMQP)
- **MapStruct** (маппинг DTO), **Lombok**
- **Flyway** (миграции БД)
- **Swagger / OpenAPI 3** (документация REST API)

### Инфраструктура & Хранение
- **PostgreSQL** — реляционная БД (хранение пользователей)
- **Redis** — in-memory хранилище (сессии, статусы асинхронных задач)
- **MinIO (S3 API)** — объектное хранилище (хранение файлов и временных архивов)
- **RabbitMQ** — брокер сообщений (асинхронная генерация ZIP-архивов)
- **Docker & Docker Compose** — полная контейнеризация всех сервисов

### Тестирование
- **JUnit 5, Mockito** — модульное тестирование бизнес-логики
- **Testcontainers** — интеграционное тестирование с поднятием реальных баз, S3 и брокеров в Docker
- **Awaitility** — тестирование асинхронных операций
- **Selenide** — E2E (End-to-End) тестирование пользовательских сценариев в Headless-браузере

### Frontend
- **React 19**, **Vite**
- **Material-UI (MUI)**

## 🚀 Быстрый запуск (Docker Compose)

Для локального запуска потребуется только установленный **Docker**  
(в среде Windows рекомендуется использовать WSL 2).

1. **Клонируйте репозиторий:**
   ```bash
   git clone https://github.com/ВашЮзернейм/Cloud-File-Storage.git
   cd Cloud-File-Storage
2. **Настройте переменные окружения:**  
   Создайте файл `.env` в корне проекта на основе примера:

   ```bash
   cp .env.example .env
Обязательно заполните данные (пароли и ключи) внутри файла .env,
а также укажите GOOGLE_CLIENT_ID и GOOGLE_CLIENT_SECRET, если хотите тестировать OAuth2.

3. **Запустите проект одной командой:**
   ```bash
   docker compose up -d --build

4. **Доступ к сервисам:**
    - 🌐 **Веб-интерфейс (Frontend):** http://localhost
    - 📚 **API Документация (Swagger):** http://localhost:8080/swagger-ui.html
    - 🪣 **MinIO Console:** http://localhost:9001 (Учетные данные из `.env`)
    - 🐰 **RabbitMQ Management:** http://localhost:15672

## 🏗 Архитектурные решения и особенности

- **Отказ от БД для файлов (Stateless)**  
  Приложение не синхронизирует состояние S3 хранилища с реляционной базой данных.  
  Вся навигация, папки и файлы извлекаются напрямую из MinIO с использованием механизма  
  S3-префиксов и разделителей (`delimiter`). Это исключает рассинхронизацию данных.

- **Асинхронность и потоковая передача (Streaming)**  
  ZIP-архивы не сохраняются на диск бэкенд-сервера (защита от переполнения диска и OutOfMemory).  
  Данные вычитываются из S3 потоком и сразу пишутся в выходной HTTP-поток  
  (для мелких файлов) или отправляются обратно в S3 через `PipedInputStream`  
  (для фоновых задач).

- **CI/CD Pipeline**  
  Настроен GitHub Actions workflow (на локальных раннерах) для автоматической сборки,  
  прогона Testcontainers / Selenide тестов и развертывания проекта при пуше в `main`.
